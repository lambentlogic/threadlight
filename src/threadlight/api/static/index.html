Weight Configuration

                                        

                                            Format: model:weight where weight is 0-100.
                                            Weights are relative - they don't need to sum to 100.
                                        

                                        

                                            Example: gpt-4o:70, claude-3-opus:30 gives 70% probability to GPT-4o
                                        

                                    

                                

                            


                            
                            

                                

                                    
                                    

                                        
Ratio Configuration

                                        

                                            Format: model:ratio where ratio is a positive integer.
                                            Ratios determine exact distribution over a cycle.
                                        

                                        

                                            Example: gpt-4o:3, claude:1 means 3 requests to GPT-4o, then 1 to Claude, repeating
                                        

                                    

                                

                            


                            
                            

                                

                                    
                                    

                                        
Order Matters

                                        

                                            Models are used in the order listed. The first model handles the first request, second model handles the second, and so on.

No routing rules defined

                                    
Add rules to route messages to different models based on content.

                                


                                
                                

                                    

                                        How it works:
                                        Rules are evaluated by priority (highest first). The first matching rule determines which model handles the message.
                                        If no rules match, the primary model is used as fallback.
                                    

                                    

                                        
Examples: "code" keyword routes to code model, "/" prefix routes to invocation model, length > 500 routes to long-context model
